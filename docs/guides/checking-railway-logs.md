# Railwayのログ確認方法

> RailwayでBotの起動ログやエラーログを確認する方法

---

## 🔍 必要なログを探す方法

### 1. 時間範囲を広げる

現在「Last 6 hr」となっている場合、もっと広い範囲に変更:

- 「Last 24 hr」を選択
- またはカスタム範囲を指定して、Botを最後にデプロイした時刻以降を選択

これで起動時のログが見つかります。

---

### 2. 検索フィルターを使う

ログ検索バー（「Filter and search logs」）で以下のキーワードで検索:

#### Bot起動時のログを探す
```
Cog 読み込み完了
```

#### スラッシュコマンド同期のログを探す
```
スラッシュコマンド同期
```

#### エラーログを探す
```
エラー
ERROR
失敗
```

#### Notebook関連のログを探す
```
notebook
Notebook
```

---

### 3. 確認すべきログメッセージ

Botが正常に起動している場合、以下のログが表示されるはずです:

```
✅ データベース初期化完了
✅ Cog 読み込み完了: cogs.onboarding
✅ Cog 読み込み完了: cogs.menu
✅ Cog 読み込み完了: cogs.vocab
✅ Cog 読み込み完了: cogs.notebook  ← これが重要！
✅ Cog 読み込み完了: cogs.svocm
✅ Cog 読み込み完了: cogs.reading
✅ Cog 読み込み完了: cogs.admin
✅ 永続 View 登録完了
✅ スラッシュコマンド同期完了（グローバル） または ✅ スラッシュコマンド同期完了（テストギルド: ...）
✅ Logged in as Winglish#1234 (123456789)
```

---

### 4. エラーログを確認

もし以下のようなエラーが出ていたら、それが原因です:

```
❌ Cog 読み込み失敗: cogs.notebook - ...
❌ スラッシュコマンド同期失敗: ...
❌ データベース初期化に失敗しました: ...
```

---

## 🚨 ログが見つからない場合

### 方法1: Botを強制再起動

1. RailwayダッシュボードでBotサービスを開く
2. 「Settings」タブを開く
3. 「Restart」ボタンをクリック
4. 再起動後、すぐにログを確認（起動ログが表示されます）

---

### 方法2: ログをリアルタイムで見る

1. RailwayダッシュボードでBotサービスを開く
2. 「Logs」タブを開く
3. Botを再起動
4. 起動ログがリアルタイムで表示されます

---

### 方法3: ログ検索のコツ

#### 複数のキーワードで検索
```
Cog AND notebook
```

#### エラーまたは失敗を検索
```
ERROR OR 失敗 OR ❌
```

#### 特定の時刻のログを検索
時間範囲を指定して、Botをデプロイした時刻以降を確認

## 💡 もしログが全く見つからない場合

### 可能性1: Botが起動していない

**確認:**
- RailwayダッシュボードでBotサービスのステータスを確認
- 「Running」になっているか確認

**解決:**
- Botを再起動
- 環境変数が正しく設定されているか確認

---

### 可能性2: ログが古すぎる

**確認:**
- 時間範囲を「Last 7 days」などに広げる
- Botを最後にデプロイした時刻を確認

**解決:**
- Botを再起動して新しいログを生成

---

### 可能性3: ログフィルターで隠れている

**確認:**
- 検索フィルターをクリア
- すべてのログを表示

**解決:**
- フィルターをリセットしてから再度確認
