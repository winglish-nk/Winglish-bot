> 現在のログは再接続ログのみ。起動ログが見えない場合は再起動が必要

---

## 🔍 現在の状況

ログを見ると、以下のメッセージしか表示されていません：

```
discord.gateway: Shard ID None has successfully RESUMED session
```

これは**Botが再接続（RESUME）しているログ**です。  
**Botの起動時のログ**（データベース初期化、Cog読み込みなど）が見えていません。

## ✅ 解決方法

### Botを強制再起動する

新しいコード（notebook Cog）を読み込むには、**Botを完全に再起動**する必要があります。

#### Railwayでの再起動方法

1. RailwayダッシュボードでBotサービスを開く
2. **「Settings」タブ**を開く
3. 下の方に**「Restart」ボタン**があるので、クリック
4. 再起動が完了するまで待つ（1-2分）

#### 再起動後の確認

再起動後、すぐにログを確認してください。以下のログが表示されるはずです：

```
✅ Cog 読み込み完了: cogs.notebook
✅ スラッシュコマンド同期完了（グローバル）
```

---

## 🔍 再起動後もログが見えない場合

### 可能性1: コードがデプロイされていない

**確認方法:**
- Railwayの「**Deployments**」タブを確認
- 最新のデプロイが成功しているか確認
- 最新のコミット（`cogs/notebook.py`を含む）がデプロイされているか確認

**解決方法:**
- GitHubでコードを確認
- 必要に応じて新しいコミットをプッシュ
- Railwayが自動的にデプロイするか確認

---

### 可能性2: ログの時間範囲が狭い

**確認方法:**
- ログ画面の時間範囲を「**Last 24 hr**」に変更
- または、Botを再起動した時刻以降を指定

---

### 可能性3: Cogの読み込みでエラーが発生している

**確認方法:**
- 検索バーで「**エラー**」や「**ERROR**」を検索
- または「**失敗**」や「**❌**」で検索

**エラーログの例:**
```
❌ Cog 読み込み失敗: cogs.notebook - ...
```

エラーが出ている場合は、エラーメッセージの内容を確認してください。
