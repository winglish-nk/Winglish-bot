# スラッシュコマンド同期ログの確認方法

ログでスラッシュコマンドが同期されているか確認する方法を説明します。

---

## 📊 正常な起動ログの確認ポイント

### 1. Cog読み込み確認

```
✅ Cog 読み込み完了: cogs.notebook
```

`cogs.notebook`が読み込まれていれば、`notebook_list_system`コマンドも定義されています。

### 2. スラッシュコマンド同期ログ

起動ログの続きに、以下のいずれかが表示されるはずです：

**グローバル同期の場合:**
```
✅ スラッシュコマンド同期完了（グローバル）
```

**テストギルド同期の場合:**
```
✅ スラッシュコマンド同期完了（テストギルド: 123456789）
```

**エラーの場合:**
```
❌ スラッシュコマンド同期失敗 (HTTP 403): ...
```

### 3. Bot起動確認

```
✅ Logged in as Winglish#1234 (123456789012345678)
```

---

## 🔍 ログが切れている場合

ログが途中で切れている場合は：

1. **時間範囲を広げる**: ログ画面で「Last 1 hour」などに変更
2. **検索で確認**: ログ検索で「スラッシュコマンド」を検索
3. **最新ログを確認**: ログ画面を最新に更新

---

## ⏰ スラッシュコマンド同期のタイミング

- **グローバル同期**: 最大1時間かかる場合があります
- **テストギルド同期**: すぐに反映されます（TEST_GUILD_IDを設定している場合）

---

## 💡 notebook_list_systemコマンドが表示されない場合

1. **コマンドが登録されているか確認**: Cog読み込みログに`cogs.notebook`が表示されているか確認
2. **同期ログを確認**: 「スラッシュコマンド同期完了」が表示されているか確認
3. **時間を待つ**: グローバル同期の場合は最大1時間かかる可能性があります
4. **TEST_GUILD_IDを設定**: すぐに反映させたい場合は、`TEST_GUILD_ID`環境変数を設定してください
